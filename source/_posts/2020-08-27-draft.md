## 修改时区

CentOS7 开始引入了一个 timedatectl 工具，直接运行命令：

```bash
$ timedatectl set-timezone  Asia/Shanghai
```

也可以手动修改时区文件： 

```bash
$ cp /usr/share/zoneinfo/Asia/Shanghai  /etc/localtime
```

如果设置时区后，时间依然快了 8 小时,可以执行下面的命令：

```bash
$ ln -s -f /usr/share/zoneinfo/Universal /etc/localtime
```

## 关闭 SELinux

临时关闭：

```bash
$ setenforce 0
```

永久关闭：

编辑 `/etc/selinux/config` 文件，将 `SELINUX` 项的值修改为 `disabled`

{% codeblock "/etc/selinux/config" lang:text %}
# This file controls the state of SELinux on the system.
# SELINUX= can take one of these three values:
#     enforcing - SELinux security policy is enforced.
#     permissive - SELinux prints warnings instead of enforcing.
#     disabled - No SELinux policy is loaded.
SELINUX=disabled
{% endcodeblock %}

参考：[查看 SELinux状态及关闭SELinux](http://blog.51cto.com/bguncle/957315)

## sudo 出现 command not found

sudo 读取的 PATH 路径与常规用户不同，而目标程序恰好没有被包含，就会出现 `command not found` 的情况。

编辑 `/etc/sudoers` 文件，将目标程序所在的路径追加上即可。

例如：

{% codeblock "/etc/selinux/config" lang:text %}
Defaults    secure_path = /sbin:/bin:/usr/sbin:/usr/bin  
{% endcodeblock %}

可以修改为：

{% codeblock "/etc/selinux/config" lang:text %}
Defaults    secure_path = /sbin:/bin:/usr/sbin:/usr/bin:/usr/local/bin:/go/bin 
{% endcodeblock %}

参考：[sudo 执行提示 Command not found](https://blog.csdn.net/u012599988/article/details/82422339)

CRI：Container Runtime Interface，容器运行时接口

## Storage Class

在 Kubernetes 中，容器挂载`卷`一般通过声明 PVC，绑定一个 PV 来实现。

PVC 由开发侧的人员声明，简单来说就是：我需要一个存储卷，你得给我，怎么实现我不管。

PV 由集群运维人员提供，PV 包含了具体的存储实现，如 nfs、ceph 等。

PVC 和 PV 是一对一的关系。

随着项目的发展，需要的存储卷也开始增加，但集群运维人员并不会以相同比例增加，此时集运运维人员就可能需要频繁的创建 PV 供相关 pod 使用。

此时，Storage Class 就出现了，其解决了 PV 需要频繁手动进行管理的问题。对于用户侧，它会根据 PVC 动态的管理 PV，不需要运维人员手动的再维护。
其底层是一个类似于存储池的实现，对于云服务提供商提供的 Storage Class，运维人员也不需要再关心存储的内容。对于自己的 Storage Class，则需要通过插件具体的存储实现。

SC 并没有破坏 PV 与 PVC 一对一的关系，SC 只是提供了一个存储池，根据需求，动态的创建 PV。PV 和 SC 底层都需要具体的存储设备支持，无论是通过云服务提供商的设备、还是 nfs、ceph 等网络文件系统，总之需要有具体的存储实现。
